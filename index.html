How It Works

Here's the code behind the demo. First, the main heading is defined by data/resume.yml. You can specify the theme color here too.

photo: "http://url/to/photo.jpg"
name: "Arya Stark"
copy: "A girl has no name..."
color: "default"
list-style: "circle"

Next, the contact info section is defined by the template located at layouts/partials/include/social.html as well as the YAML front-matter located at data/social.yml. You can use Font Awesome icons.

<p>
  <i class="fab fa-github fa-lg fa-fw"></i>
  <a href="https://github.com/{{ . }}"><i class="fas fa-at"></i>{{ . }}</a>
</p>

...

Finally, the cards are defined by markdown files under the content/post directory. These files also use YAML front matter.

---
type: "Work Experience"
icon: "laptop-code"
heading: "Zone Laser Tag"
subheading: "Intern, Software Development"
duration: "May 2011 – July 2011"
---

Summer Intern for the team located in Dover, Delaware. Assisted in development of Python-based software applications.
...

These files are put together by Hugo, which is supported by GitHub Pages.

Also, card heights are fixed by CSS. So, if all the texts fit within each card when viewed on the web, they'd also fit within each card when clicking the "Saved as PDF" button.

You can read more detailed documentation on GitHub.
